🔧 LOCAL SETUP (5 MINUTES)

1. Extract files to a folder
2. Open terminal in that folder
3. Run these commands:

   npm install
   cp .env.example .env
   
4. Edit .env and add:
   SHOPIFY_API_KEY=your_key
   SHOPIFY_API_SECRET=your_secret
   SHOPIFY_REDIRECT_URI=http://localhost:3000/callback

5. Start server:
   npm start

6. Open browser: http://localhost:3000
7. Connect your test Shopify store

════════════════════════════════════════════════════════════════

🌐 DEPLOY TO RENDER (10 MINUTES)

Step 1: Push to GitHub
  git add .
  git commit -m "Deploy to production"
  git push origin main

Step 2: Go to Render.com
  → New → Web Service
  → Connect GitHub
  → Select this repo

Step 3: Configure
  Name: shopify-saas
  Runtime: Node
  Build: npm install
  Start: npm start

Step 4: Environment Variables
  SHOPIFY_API_KEY=...
  SHOPIFY_API_SECRET=...
  SHOPIFY_SCOPES=read_products,write_products,read_orders,read_inventory,write_inventory,read_customers,write_customers
  SHOPIFY_REDIRECT_URI=https://[your-render-url].onrender.com/callback
  PORT=3000
  NODE_ENV=production

Step 5: Deploy & Wait (2-3 minutes)
  → Once "Live" appears, you're done

Step 6: Update Shopify App Settings
  → Shopify Partners Dashboard
  → Settings → Redirect URIs
  → Change to: https://[your-render-url].onrender.com/callback

🆘 QUICK TROUBLESHOOTING

Problem: "Shop not connected"
→ Check SHOPIFY_API_KEY & SHOPIFY_API_SECRET in .env
→ Verify SHOPIFY_SCOPES are set
→ Clear browser cookies

Problem: "OAuth callback failed"
→ Confirm SHOPIFY_REDIRECT_URI matches in .env AND Shopify settings
→ Check URL is HTTPS in production

Problem: "Render deployment stuck"
→ Check Render Logs for errors
→ Verify all env variables are set
→ Redeploy: git push origin main

Problem: "Dashboard shows no data"
→ Ensure store has orders/products
→ Check browser console for API errors
→ Verify API scopes in Shopify App settings
